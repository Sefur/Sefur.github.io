git的安装与使用

由于工作需要用到版本控制工具，首先想到的就是git。理由有这么两个：

  1. 免费且强大
  2. 大名鼎鼎，使用者不计其数，文档资料完善

由于我是用的win10系统，直接官网下载安装git就可以了。一般介绍的是git命令行，至于gui，那就看官方文档了。

然后我就可以在本地看到一个项目的文件夹，使用git命令行到该文件夹下，可以看到目前所处的分支(branch)。
由于开发一个项目工程，需要保证之前所做的工作不会被后面的修改所影响，分支应运而生，项目中添加新的特性可以在分支上增加、删除、修改，使劲折腾，完全不会影响主分支(master)上原有的代码。当新的东西测试验证后，就可以放心大胆的合并(merge)到master了。

有一个非常类似的命令 git pull，这个命令相当于将远程仓库更新到本地，有一个merge的过程，而fetch则没有。当我们在某一个分支上准备推送(push)代码到remote时，可能往往需要pull一下本分支，将本分支更新到最新才能push，这就可能带来merge(本地准备push的分支不是最新的分支，并且我们修改了并提交了里面的代码,此时同一个文件中远程和本地的内容不一样，或者存在冲突，或者远程端其他文件有修改，我们本地这份没有，就需要我们来进行merge了) 过多的merge会带来一个问题，多人同时在一个分支上作业的话，最终各自push,假设每个人修改项目中一个文件，其他人并不重复，最后依次push的时候，后一个人就需要先pull最新分支，根据上一段提到，此时会产生merge。这样下来，显得比较混乱，我们可以使用以下命令消除
